													TestNG Notes
----------------------------------------------------------------------------------------------------------------------------------------------------------------
01. TestNG execute test method based on alphabetical order
02.@Test (Priority=num) controls the order of execution
03. Once you assign the priority on the methods then the order of method is not considered 
04.Priority can be any random number (no need to be consecutive number)
05 if you don't provide the priority then the default values will be Zero(0)
06.if the priority are same for 2 methods then it take default priority and execute 
07 Negative values are allowed in TestNG priority 
08.TestNG execute test methods only if they are having @Test annotation

TestNG Listner 
---------------------------------
Listner are the class which will run after the Test Method run 


Listner class can be used in two way 

1.By implementing the ITestListner interface
Class  myListner implements ITestListner{

}
2.By extening the TestListnerAdapter
Class myListner extends TestListnerAdapter{

}


we can run the Lister and Test Case both by using xmal file 

<suite name="Suite">
<listeners>
<listener class-name="raju.selenium.myListner"></listener>
</listeners>
  <test thread-count="5" name="TestNGListner">
    <classes>
      <class name="raju.selenium.TestNGListner"/>
    </classes>
  </test> <!-- TestNGListner -->
</suite> <!-- Suite -->

if you do not want to run the Listner by using xaml the  you can do that by using TestNG annotation
on Test Case class 

@Listner(packagename.ClassName.Class)
Classs TestCase{
	Void TestMethod1(){

}
}

Page Class
--------------
public class loginPage2 {
	
	WebDriver driver;
	loginPage2(WebDriver driver){
		this.driver=driver;
		PageFactory.initElements(driver, this);
	}
	
	@FindBy(xpath = "//input[@placeholder='Username']") WebElement txt_UserName;
	@FindBy(xpath = "//input[@placeholder='Password']") WebElement txt_password;
	@FindBy(xpath = "//button[normalize-space()='Login']") WebElement Login_button;
	
	
	public  void setUserName(String userName) {
		txt_UserName.sendKeys(userName);
	}
	public void SetPassword(String passwrd) {
		txt_password.sendKeys(passwrd);
	}
	public void ClickLoginButton() {
		Login_button.click();
		
	}
}
Other way to use page object is 
------------------------------------
public class loginPage {
	//COnstructor 
	WebDriver driver;
	
	loginPage(WebDriver driver){
		this.driver=driver;
	}
	
	//Locator 
	By txt_UserName=By.xpath("//input[@placeholder='Username']");
	By txt_password=By.xpath("//input[@placeholder='Password']");
	By btn_loginbutton=By.xpath("//button[normalize-space()='Login']");
	
	//Action Methods
	
public void setUserName(String userName) {
	driver.findElement(txt_UserName).sendKeys(userName);
}
public void setPassword(String password) {
	driver.findElement(txt_password).sendKeys(password);
}
public void clickLoginButton() {
	driver.findElement(btn_loginbutton).click();
}
	
}
